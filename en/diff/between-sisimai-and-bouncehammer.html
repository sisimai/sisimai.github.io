Resolved Problems

The final version of {{ site.html.bh }} (v2.7.13) had several critical and difficult-to-solve problems related to analysis accuracy. {{ site.html.ss }} was developed to address these problems.

2 or more bounces in a single message

Perl
Ruby

{{ site.html.ss }} can detect all recipient addresses and error reasons from a single message which include 2 or more bounce records, such as those listed below. {{ site.html.bh }} can only detect the first one.

Bounce mails in Gmail mailbox

Perl
Ruby

{{ site.html.ss }} can also decode bounce emails in your Gmail mailbox. {{ site.html.bh }} can also decode bounce emails in your Gmail mailbox if they are in an older format, but it cannot decode bounce emails in the current format.

Perl
Ruby

{{ site.html.ss }} can correctly decode bounce emails (e.g., rfc3464-28.eml ) that contain lines like Status: 2.1.5 and determine the bounce reason as "delivered". However, {{ site.html.bh }} can also decode such emails, but it determines the bounce reason as "undefined", as shown in the output example below.

Feedback Loop message

Perl
Ruby

{{ site.html.ss }} can also decode emails in the ARF (Abuse Reporting Format) format, which is used by overseas email services such as AOL and Yahoo!. {{ site.html.bh }}'s FBL support is only an experimental implementation and cannot fully decode emails in the ARF format.

{{ site.html.ss }} works on Perl and Ruby

Perl
Ruby

{{ site.html.ss }} works with Perl {{ site.sisimai.workson.perl[0] }} and later, while {{ site.html.bh }} does not work with Perl 5.16 and later. Therefore, if the Perl on the server where you are trying to install {{ site.html.bh }} is 5.18, you will need to install an older version of Perl separately.

And, from {{ site.html.ss }} v4.14.2, {{ site.html.ss }} works on both Perl and Ruby (including JRuby).

Easy to install

Perl
Ruby

Installing {{ site.html.bh }} is a bit of a hassle, and it requires downloading the archive, installing many dependent modules, and running the configure, make, make test, and make install commands. On the other hand, {{ site.html.ss }} can be installed with the cpanm or gem commands, making it very easy to install.

Command Line Tools, DB Schema, WebUI

Perl
Ruby

{{ site.html.ss }} does not have command-line tools like {{ site.html.bh }}'s bin/mailboxparser and bin/databasectl, nor does it have a WebUI or DB schema. It is provided only as a lightweight and simple module. The idea is that you can implement command-line tools as you like with a few lines of code, or integrate it into your existing system.

High Precision of Analysis

Perl
Ruby

{{ site.html.ss }} has {{ site.sisimai.spec.modules }} built-in MTA/ESP decoding modules, giving it about twice the analysis accuracy of {{ site.html.bh }}. {{ site.html.ss }} can also decode bounce emails from commercial MTAs such as Microsoft Exchange Server and IBM Notes and Domino, as well as commercial email services such as Amazon SES, WorkMail, and SendGrid.

Faster than {{ site.html.bh }}

Perl
Ruby

The Perl version of {{ site.html.ss }} decode emails and generates structured data about {{ site.sisimai.diff.timesfaster.perl }} times faster than the {{ site.html.bh }} bin/mailboxparser command.

Small and Robust

Perl
Ruby

{{ site.html.ss }} is a library that specializes in decoding bounce emails, and therefore it is designed to be small and robust, as follows:

  • About half the number of lines of source code as {{ site.html.bh }}
  • About five times the number of tests as {{ site.html.bh }}
  • Only two dependent modules
