What is {{ site.html.ss }} ?

{{ site.html.ss }} is a library that decodes complex and diverse bounce emails and outputs the results of the delivery failure, such as the reason for the bounce and the recipient email address, in structured data. It is also possible to output in JSON format.

The latest stable version is {{ site.html.ss }} {{ site.sisimai.version }} , released on {{ site.sisimai.release.en }}. The Go version of {{ site.html.ss }} is also available from this version.
For the Perl and Ruby versions, the changes include renaming the item names ("smtpagent" and "smtpcommand") of the parsed data due to maintaining compatibility with the Go version, adding a new data item ("feedbackid"), creating two new bounce reasons ("FailedSTARTTLS" and "Suppressed"), and changing other internal APIs. For details on the all changes in Sisimai {{ site.sisimai.version }}, please see the release notes.
Please check Differences with {{ site.html.ss }} 4 page for the breaking changes in {{ site.html.ss }} 5.
{{ site.html.ss }} versions 4.25.14p11 and earlier contain a regular expression vulnerability (ReDoS: CVE-2022-4891). If you are using one of these versions, please upgrade to v4.25.14p12 or later.

About the Logo only showcase

vs. {{ site.html.bh }}

In the summer of 2014, Perl version {{ site.html.ss }} was developed to solve numerous critical issues related to the parsing accuracy of {{ site.html.bh }}. Two years later, on February 3, 2016, the Ruby version of {{ site.html.ss }} was also released.

Both the Perl and Ruby versions of {{ site.html.ss }} continue to parse 110 million bounce emails worldwide every month (according to our research). On February 2, 2024, {{ site.html.ss }} 5.0.0 was released, capable of detecting modern bounces, including domain authentication (SPF, DKIM, DMARC, ARC) errors and reputation-related errors. On August 16 of the same year, it celebrated its 10th anniversary since its release.

And on February 25, 2025, Go version {{ site.html.ss }}, an implementation in a non-scripting language, was released as the third {{ site.html.ss }}. Go version {{ site.html.ss }}, like the Perl and Ruby versions, minimizes dependencies and is characterized by the ease of deployment and execution speed ( {{ site.sisimai.diff.timesfaster.golang }} times faster than bounceHammer, {{ site.sisimai.diff.fasterthan4.golang }} times faster than Perl version {{ site.html.ss }} v5 ), which are advantages of compiled binaries.
