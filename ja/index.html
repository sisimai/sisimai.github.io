{{ site.html.ss }}(シシマイ)は複雑で多種多様なバウンスメールを解析してバウンスした 理由や宛先メールアドレスなど、配信が失敗した結果を構造化データで出力するライブラリで JSONでの出力も可能です

現在の安定版最新バージョンは{{ site.sisimai.release.ja }}にリリースした {{ site.html.ss }} {{ site.sisimai.version }}です。 主な変更点は、Ruby版の間違った型に起因するひどいバグの修正と Gmailにおける変更されたエラーコード対応です。 変更点の詳細はリリースノート( v{{ site.sisimai.version }} )をご覧ください。
{{ site.html.ss }} 5で入った非互換な変更点はSisimai 4との違いを確認してください。
{{ site.html.ss }} 4.25.14p11およびそれ以前のバージョンには 正規表現に関する脆弱性(ReDoS: CVE-2022-4891 )があります。 お使いの場合はv4.25.14p12以降へアップグレードしてください。

使ってみる！

{{ site.html.ss }}はUNIX mbox形式のファイルや Maildir/ディレクトリ内にあるような バウンスメール用解析ライブラリです。PerlまたはRubyで動作します。

ロゴ表示について

vs. {{ site.html.bh }}

2014年夏、Perl版{{ site.html.ss }}(シシマイ)は {{ site.html.bh }}が抱える解析精度に関わる致命的な問題の数々を解決する為に開発されました。 その二年後、2016年2月3日にはRuby版{{ site.html.ss }}もリリースされました。

Perl版とRuby版の両{{ site.html.ss }}は世界で毎月1.1億通(弊社調べ)のバウンスメールを解析し続け、 2024年2月2日にはドメイン認証(SPF, DKIM, DMARC, ARC)系エラーやレピュテーションに起因するエラーなど 現代のバウンスも検出可能な{{ site.html.ss }} 5.0.0をリリースし、 同年8月16日にはリリースから10周年を迎えました。

そして2025年2月25日、第三のシシマイとしてスクリプト言語ではない実装であるGo版{{ site.html.ss }} がリリースされました。Go版{{ site.html.ss }}はPerl版やRuby版と同様に依存も最小限に抑えられ、 コンパイル済みバイナリの利点であるデプロイの容易さや実行速度 ({{ site.html.bh }}の{{ site.sisimai.diff.timesfaster.golang }}倍、Perl版{{ site.html.ss }} v5 の{{ site.sisimai.diff.fasterthan4.golang }}倍)が特徴的です。

サポート

{{ site.html.ss }}はオープンソースライセンス(二条項BSD) で公開しているライブラリですので、どなた様でも自由にお使い頂けます。
しかし、バウンスメール解析の為の環境構築や {{ site.html.ss }}周辺スクリプトの開発、解析済データを蓄積するデータベース設計や管理、 継続的なバウンス管理、そしてこれまでのバウンスメール解析システムである {{ site.html.bh }}からの移行などをサポートする品目を、 開発元である株式会社{{ site.html.cr }}にて提供しております。
